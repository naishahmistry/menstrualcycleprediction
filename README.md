# Menstrual Cycle Phase Prediction & Symptom Forecasting

## Overview
Despite the rise of period tracking apps like Flo and Stardust and the growing focus on women’s health in the health tech industry, research on AI/ML applications for addressing menstruation, PCOS, and endometriosis is still limited. The majority of period tracking apps use calendar-based methods to predict future cycles based on previous cycle lengths, but they disregard the nuances that come from physiological and lifestyle signals, such as skin temperature, heart rate, sleep, and stress. Similarly, while wearables collect physiological data, these systems have been built on male-dominant data that fail to account for the variability and cyclical nature of female biological systems. Accurate menstrual cycle tracking and prediction is significant since it not only informs reproductive health decisions and fertility planning, but it also improves an individual’s quality of life by enabling them to anticipate and circumvent adverse symptoms or major health risks. Existing research has made meaningful progress in menstrual cycle phase prediction using a variety of machine learning approaches and factors of interest, ranging from using Basal Body Temperature (BBT) with state-space models to using BBT, sleep heart rate, and time since start of menstruation using XGBoost models. However, most of this work uses a relatively limited set of physiological symptoms, and there does not appear to be much research into symptom prediction within each phase. This project aims to bridge this gap by using Bayesian Machine Learning and neural networks to leverage physiological and lifestyle data collected from wearables from the [mcPHASES Dataset](https://physionet.org/content/mcphases/1.0.0/) to detect an individual’s cycle phase from biometric signals and forecast symptom severity.  

## Key Components
### Bayesian Multinomial Logistic Regression for Cycle Phase Classification 
We aim to expand on existing research by building a Bayesian Multinomial Logistic Regression model to predict the menstrual cycle phase (Fertility/Ovulation, Follicular, Luteal, and Menstruation) based on a larger range of input features that encompass biometric signals, lifestyle, and hormone levels (LH and Estrogen). This model will consider physiological signals like heart rate and glucose levels, as well as sleep quality/duration, stress levels, and physical activity. 

### Multi-Layer Perceptron (MLP) for Symptom Forecasting 
To further our project into a valuable tool for individuals to use on a regular basis, we aim to build an MLP that will forecast whether an individual will experience common symptoms associated with menstruation. An MLP is appropriate for this use case since we can utilize the sigmoid activation function to output the likelihood of a symptom occurring while considering multiple biometric and hormonal inputs, and it can also be extended by using alternative activation functions to solve multi-class problems, such as predicting the severity of these symptoms. The symptoms that we will predict include appetite, headaches, cramps, sore breasts, fatigue, sleep issues, mood swings, stress, food cravings, indigestion, and bloating.

To predict symptoms that individuals typically experience during the menstrual cycle, such as cramps, headaches, fatigue, and bloating, we will initially test a Multi-Layer Perceptron with the architecture above and Binary Cross-Entropy Loss. This model will start with a 12x1 input vector that will represent the following: whether the individual is in the Menstrual phase, whether the individual is in the Luteal Phase, whether the individual is in the Fertility/Ovulation phase, whether the individual is in the Follicular Phase, the day in the study, LH levels, estrogen levels, mean resting heart rate, sleep efficiency mean, overall sleep score, mean glucose, and active minutes. We aim to predict eleven target symptoms: appetite, headaches, cramps, sore breasts, fatigue, sleep issues, mood swings, stress, food cravings, indigestion, and bloating. While the data encodes the symptom results categorically (Not At All, Very Low/Little, Moderate, High, Very High), we will initially convert these data points into binary outputs where “Not At All” indicates that the individual does not have the symptom and any other value indicates that the individual has the symptom. Our starting architecture will use a ReLU activation function across all of the hidden layers and use Sigmoid as the final activation function to ensure a binary output, and we will use Binary Cross Entropy Loss to evaluate the accuracy of the model. The current architecture is designed to only predict one symptom at a time, but we hope to expand this to have an eleven unit output to support all symptoms in one MLP since they will be predicted using the same inputs. We also hope to expand this to support multi-class classification by using alternative activation and loss functions.
